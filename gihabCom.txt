
git status # Проверка статуса

git add . # Добавление изменений

git diff # Проверка изменений

git log # Проверка истории

git commit -m "редактирование README.md"

git push origin main # Отправка изменений на удаленный репозиторий

git pull origin main # Получение изменений с удаленного репозиторияs

git stash # Сохранение изменений

Рабочий цикл между машинами:
На Машине 1:
1.
git clone <URL> (только один раз для получения проекта).
2.
Внесите изменения.
3.
git status -> git add . -> git commit -m "...".
4.
git push.
На Машине 2:
1.
git clone <URL> (только один раз).
2.
git pull (чтобы получить изменения с Машины 1).
3.
Внесите изменения.
4.
git status -> git add . -> git commit -m "...".
5.
git push.
На Машине 3:
1.
git clone <URL> (только один раз).
2.
git pull (чтобы получить изменения с Машин 1 и 2).
3.
Внесите изменения.
4.
git status -> git add . -> git commit -m "...".
5.
git push.
И так далее. Перед началом работы на любой машине всегда делайте git pull, 
чтобы убедиться, что у вас последняя версия. После завершения работы или 
достижения логической точки, делайте git add, git commit, git push, чтобы 
сохранить и синхронизировать ваши изменения.
